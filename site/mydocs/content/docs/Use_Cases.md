# Use Cases

Use Cases

Where would you use Kagura:

    When you need a reporting web service that is quick to deploy and more developer friendly than existing frameworks

    When you need something that you can customize as all levels

    When you need something you can embed into your application or run separately, with no changes required to port your reports across

    When you need a report service which you can run in a stateless environment
